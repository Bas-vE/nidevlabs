The Diagram SDK Overview

The NI Diagram SDK is a collection of .NET base classes, interfaces, and APIs designed to help you build:

  • Diagrammatic models of computation (MoCs)
  • Graphical MoC editors

Diagram SDK Theory of Operation

The Diagram SDK consists of three primary components:

  1. The shell provides a graphical user interface framework for building graphical MoC editors. The shell includes the following sub-components:
  2. The document system manages, displays and edits the end user's source files.
  3. The diagram editing system displays and edits source file documents as graphical diagrams.
  4. The diagram object model (DiOM) provides a hierarchy of object-oriented classes and interfaces for modeling diagrammatic MoCs. The DiOM includes the following sub-components:
  5. The transaction system manages changes to DiOM Elements.
  6. The name resolution system allows diagrams to link to external resources, such as classes and methods defined in a separate assembly.
  7. The persistence system saves and loads your MoC editor's diagrams as XML documents.
  8. The messaging system collects messages about DiOM Elements, such as compiler errors and warnings.
  9. The wiring system manages wiring between nodes on a diagram.
  10. The plug-in system allows you to write classes and methods that plug into the shell and the DiOM.

Follow the Diagram SDK tutorials to learn more about using the Diagram SDK to create a simple graphical MoC editor.
