Remarks

This synchronously returns the resolved envoy if one can be found. It should only be called on a cached dependency belonging to a cache service that's already initialized. Otherwise, you should use ResolveAsync() instead.

By default, we just resolve to envoys with the given target name. Clients should override this property if they want to further filter the list of eligible envoys to resolve to based on criteria other than just the envoy name. If multiple envoys resolve based on the name, the dependency is ambiguous and null will be returned. Clients should override this property if they want a different behavior when dealing with ambiguous references.
