Enum QuantizationMode

Represents the quantization mode that should be used for fixed-point computations. Quantization is the way that data should be transformed when the computation actually produced precision that is different than what can be represented by the data type itself.

Namespace: NationalInstruments.DataTypes
Assembly: NationalInstruments.PlatformFramework.dll
Syntax
public enum QuantizationMode : short

Fields

Name Description
ConvergentRounding

Round to nearest, with ties rounded to the even value. Also known as convergent, banker's, or unbiased rounding. This is the IEEE-754 standard mode for floating-point operations.
Default

The user has not set the value for this operator, use the value otherwise specified

RoundToNearest

Round to nearest, with ties rounded toward positive infinity. Also known as grade school rounding. Introduces a slight positive bias.
RoundTowardPositiveInfinity

Round toward positive infinity. Also known as ceiling. Introduces a positive bias.
ShouldNotQuantize

Don't do anything. Just keep the original bits of the data.
Truncate

Round toward negative infinity. Also known as floor, or truncate for integer or fixed-point numbers. Introduces a negative bias.
TruncateToZero

Round toward zero. Also known as truncate for floating-point or sign magnitude numbers. This is equivalent to for unsigned numbers. Introduces a negative magnitude bias.
