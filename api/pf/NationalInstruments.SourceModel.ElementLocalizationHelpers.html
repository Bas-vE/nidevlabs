GetLocalizedAttributeResourceKey(Element, PropertySymbol, String)

Given an element, create the key it uses in a localized strings dictionary to locate its value.

Declaration
public static string GetLocalizedAttributeResourceKey(this Element element, PropertySymbol symbol, string keyPrefix)
Parameters
Type Name Description
Element element

The element whose property symbol needs a localization key.
PropertySymbol symbol

The property symbol for which a key is needed.
String keyPrefix

A prefix to place in front of the calculated key.
Returns
Type Description
String

The key in the localized strings dictionary that should locate the necessary localized string.
Remarks

Key generation follows a relatively simple set of rules. Without having an exhaustive hierarchical path of the Element's ownership, it tries to have a reasonable key that is somewhat human-readable. The key will always contain the property symbol and the XML LocalName of the Element with which the symbol is associated as its last two "elements". The key may be further augmented by Element-specific names via the LocalizationKeyBaseName property, which helps to disambiguate which Element the key is associated with. For example, for the Label text of a button having the label "MyButton", the key is: Panel_MyButton_Label_Text In this key, the last two "elements" consist of the PropertySymbol's name ("Text"), the Element to which it belongs ("Label"), the LocalizationKeyBaseName provided by the Button (in this specific case, the user-facing label text itself), and finally the "Panel", provided by the VI's panel itself. Each "element" in the label is separated by an underscore character. Also, although the LocalizationKeyBaseName text may be arbitrary, it will be 'sanitized' by replacing or encoding unsupported characters, as the key may be used in XAML.

Exceptions
Type Condition
InvalidOperationException

Thrown if an error occurs attempting to create the dictionary key.
