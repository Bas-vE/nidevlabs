Class DocumentEditControl

The DocumentEditControl is the root visual element for all editors for a Document. Its Content is usually a DesignerEditControl for PlatformFramework model documents but can be any FrameworkElement.

DoInitialize()

This method performs the onetime setup that is required that may be dependent on visual tree being ready. It will not do work if we have already been initialized. PJC - API - this is confusing as our base class (FrameworkElement) also has an Initialized state/event...which is different than this. Key use cases:

  1. Let the FP and Icon Editor initialize their panels after load.
  2. Let the Root Diagram perform onetime initialization once it is ready to go. Ready to go means that: A. It has it's DataContext so it can bring it locally B. It is a descendant of DesignerEditControl (i.e. completely in the LogicalTree so RoutedEvents fired during binding work. C. It can get AttachedProperties (SourceFile, EditorName, ... so that it can initialize Dynamic Resource Dictionaries
