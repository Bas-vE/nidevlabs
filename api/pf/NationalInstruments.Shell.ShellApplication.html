UILanguage

Gets the UI language actually used for display. This may be different from CulureInfo.CurrentUICulture.Name because that is defined either by user settings (via explicit choice), or implicitly by the OS. For example, on a Windows installation running in Spanish, if the user does not explicitly set our application's UI language, CultureInfo.CurrentUICulture.Name may be es-ES. However, if Spanish resources are not available, the UI will display English strings. This property will return the language actually used for display, in this case, en-US.
