OnDetach(Envoy)

Called to detach this envoy service from its AssociatedEnvoy. This method can be called when the associated envoy is being removed from the project, or when the associated envoy changes in such a way that this service no longer binds to the associated envoy.

If the associated envoy is being removed from the project, it is possible that other envoy services bound to this associated envoy will have already been detached. Therefore, it is not safe to query for other services within this callback. Instead, this callback should only be used to clean up state that is local to and understood by this service.

Remarks

This method is only called when the model referenced by the Envoy is closed while the envoy is still attached. This usually only happens when the last reference to the referenced model is released. When the last reference is released, the model is unloaded and can be loaded again later by calling LoadAsync(). If the envoy itself is removed from the project, this method will not be called.
