Class ModelSideEffects

Helpers to apply a "side effect" transaction. A side effect transaction is a modification to a source model in response to a modification of another source file. This is hard because of the locking rules where you cannot acquire a write lock while holding another lock. These helpers deal with this rule by implementing an asynchronous inspection and application strategy. Basically the time line looks like this:

Receive model change notification Schedule side effect

  • await for changed model to be unlocked
  • acquire upgradeable read lock on target model
  • acquire read lock on changed model Inspect - Both the modified and target models are read locked and can be accessed
  • release read lock on changed model
  • upgrade to write lock
  • begin transaction Apply changes to target model based only on inspection results
  • commit transaction
  • unlock everything

These helpers will handle transacting in the correct thread There are several patterns of inspection and apply changes that are all lock and thread correct
