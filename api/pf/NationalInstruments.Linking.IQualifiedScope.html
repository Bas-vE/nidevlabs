Interface IQualifiedScope

The IQualifiedScope is an interface implemented on objects that resolve QualifiedNames into Envoys.

Remarks

A IQualifiedScope is a node of a tree of scopes. The scopes are rooted on a global scope. The process of resolving a QualifiedName into a Envoy always searches all scopes in a hierarchy. The reason for searching all scopes is so that clients can detect when a single name resolves to two or more Envoys. It goes without saying that a scope must not prevent two Envoys from having the same name.

The IQualifiedScope interface doesn't include a name property. This means a scope by default is anonymous. Envoys in an anonymous scope can only be found by a resolve within the scope. A scope is named when the type implementing IQualifiedScope also implements Envoy.

A scope can be partial. A partial scope is a named scope that is spread across multiple IQualifiedScopes siblings. A partial scope is useful when the logical scope will be split across multiple objects that implement IQualifiedScope. An example of a partial scope is a C# namespace. A C# namespace can exists across multiple files but all types within the namespace can resolve the types in the namespace.

AutomaticallyResolveUp

Returns true if the Linker should automatically resolve up through the scope and false otherwise.

Example 1: In the given hierarchy, if ChildScope automatically resolves up, ChildScope.Resolve("TargetEnvoy") will resolve to both :Project:ParentScope:ChildScope:TargetEnvoy and :Project:ParentScope:TargetEnvoy. But if ChildScope did not automatically resolve up, it would only resolve to :Project:ParentScope:ChildScope:TargetEnvoy.

Project ParentScope TargetEnvoy ChildScope TargetEnvoy

Example 2: In the given hierarchy, if ChildScope automatically resolves up, ChildScope.Resolve("SubScope:TargetEnvoy") will resolve to both :Project:ParentScope:ChildScope:SubScope:TargetEnvoy and :Project:ParentScope:SubScope:TargetEnvoy. But if ChildScope did not automatically resolve up, it would only resolve to :Project:ParentScope:ChildScope:SubScope:TargetEnvoy.

Project ParentScope SubScope TargetEnvoy ChildScope SubScope TargetEnvoy
