CopyHierarchy(IReferencedFileService, String, String, String, CopyHierarchyComplete)

Create an in memory, unsaved, deep copy of the given hierarchy into the same name scope as the original hierarchy. Each item in the hierarchy will have a name based on the original name combined with a prefix and/or suffix. All dependencies in the copy will refer to the copy and not the original other than repository manager items.

System.Action<Exception, System.Collections.Generic.IList<Envoy>, System.Collections.Generic.IList<EnvoyError>> operationComplete

Callback when the operation is complete

  • Exception Parameter - If an exception occurred in the rename framework, it is reported here
  • IList{Envoy} Parameter - The envoys that were successfully updated
  • IList{EnvoyError} Parameter - The envoys that failed to update and the associated exception
Remarks

This is an asynchronous method. However, it is not safe to allow just any operation to run during this method. The caller of this method should likely block the UI thread (possibly through a dialog).
