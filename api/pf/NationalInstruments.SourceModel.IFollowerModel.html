Interface IFollowerModel

Interface to define follower models. A follower model means it might not exist independently. It will belongs to an other model if Leader is not null.

The follower model definition is not limited to first drop selection use case. It is a general concept of indicating whether a model can exist independently. If its leader is not null, it means it is part of other model.No matter whether the leader / follower is side by side or one contains the other. Base on this definition, Below is some example help to understand the concept:

  1. Tab item is part of tab control when it is used by a tab control.
  2. Case selector is part of structure when it is used by a structure as well as Loop Count, Nested Diagram and Structure builtin.
  3. Border node has wider scale than the specified border node like case selector and loop count. For example, the tunnel actually depends on the wire connect with it and the structure holds it. So it is hard to say who is the leader. So I avoid to put this concept into more generic class.
