Syntax
public abstract class Element : IPropertyLocalStorage, IParsable, IQueryService
Remarks

Every object in a model is an Element. The elements of a model are related to other elements through the Owner and Components properties. The sequence of owner elements from one element to another are the element ancestors. All elements reachable through the elements Components and their children are the descendants. The ancestor and descendant relationships are important since a number of features rely on finding services through their ancestors or descendants.

An element in the model must report changes to the transaction system. An element reports changes to itself through a ITransactionRecruiter available through the TransactionRecruiter property. A client changing an element must use its ITransactionManager through the property TransactionManager if the element has a transaction manager. If the TransactionManager property is null then you obviously don't need to use it.

The GetMessages() collection property stores messages that indicate errors or warnings regarding the state of the element. An element in a hierarchy of elements exposes its descendant error messages through MessageScope collection.

An element can be valid or invalid. A valid element has a consistent state and relationship with other elements. A element's validity is determined by calling the Validate(ValidationContext) method. The Validate method invokes CheckModel(ValidationContext). Derived types should override the CheckModel to enforce the the derived types restrictions.

ParsableNamespaceVersionCompatibility

public virtual IEnumerable<ParsableNamespaceVersionCompatibility> ParsableNamespaceVersionCompatibility { get; }
Property Value
Type Description
System.Collections.Generic.IEnumerable<ParsableNamespaceVersionCompatibility>
Remarks

This information is used to build up the SourceModelFeatureSet when persisting model elements. The namespaces and versions returned will be combined into a single table for all Elements that are being persisted. If multiple elements require the same namespace, the highest requested version for the namespace will be stored in the table. If you don't want the element to override the OldestCompatibleVersion, you can return VersionZero.
By default, this function will just return the namespace name defined by XmlElementName and an oldest compatible version of VersionZero. If the element can contain properties with namespaces that differ from that of the element, you should override this method to return the version information for the namespaces of the properties in addition to the version information for the element's namespace. If the element derives from an elment in another namespace or if the base class declares properties from another namespace, you should chain this call to the base class so the base class can also record its namespace and version requirements.

TransactionManager

Gets the ITransactionManager for a hierarchy of elements.

Declaration
public ITransactionManager TransactionManager { get; }
Property Value
Type Description
ITransactionManager
Remarks

An element may or may NOT have a ITransactionManager. An element has a transaction manager when it is attached to a SourceFile or Definition that has a transaction manager. An element can still be modified without a transaction manager. The modifications to an element without a transaction manager will be stored up until it can get access to a transaction manager. Any changes made to the object prior to attaching to a transaction manager are merged into the current transaction. If an element has a transaction manager than it can only be changed when a transaction is active. See ITransactionManager for how to begin transactions.

Gets the ITransactionServices object that was provided to this Element.
Usually, getting the properties (and calling the methods) on this interface are exactly the same as getting the corresponding properties of the Element object. However, when this object also implements ITransactionServices, the services that are provided to this object will be provided through this property, instead of this object itself. However, the properties directly on Element will always return the values for the object itself.

For instance, on SourceFile, TransactionManager will property return the transaction manager owned by the source file, and used to transact changes on all of its elements.
