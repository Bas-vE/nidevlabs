LayoutCompleted

When we resolve, we kick off an async task which ends up updating the terminal layout of the sourcemodel node (amongst other things). However, SelectedDependencyTargetExport has already been updated so the resolve is complete even though its after effects may not be. If LayoutCompleted is false, then we know that we are in this time where we have resolved but the layout has not yet been updated.

EnsureStateFromQualifiedTargets()

This function is called to update the source of the dependency against the current state of the resolved target. It should verify that if its target is still there (i.e. has not been removed from its SourceFile, which would have called EvaluateUnresolve prior to this call), the side-effects done by EvaluateResolve are still in place.

If any side-effect transactions are done in EvaluateNameChange, those also must be re-applied here.
