CommitAndDiscardUnsafely()

Commits the transaction without adding it to the undo stack. If the transaction is marked to propagate to linked contexts, it will still get propagated. The propagated transactions will also be discarded on commit in the linked context.

Declaration
public ICompletedTransaction CommitAndDiscardUnsafely()
Returns
Type Description
ICompletedTransaction

the committed transaction
Remarks

This should only be used with great care and caution as it can easily lead to crashes and unexpected results. It's often difficult to know exactly what changes are present in a transaction when discarding and if any of the objects modified in the discarded transaction have also been modified in transactions already committed to the undo stack. By discarding this transaction, those objects will no longer be in the same ending state they were expecting when/if the transaction is undone. Assuming the undo can tolerate this change in state and behaves sanely, an undo/redo around the discarded transaction could still leave the objects in a different state than when the transaction was discarded. Often times discarding the entire undo stack is a safer alternative than trying to selectively discard a single transaction.
