Remarks

Caution for types that derive from Element instead of Content... This base class does not automatically record a transaction for it's own creation, like does. This means that you need to be very careful about ordering of transacted changes, particularly before the element is attached to an owner hierarchy. One strategy is to avoid transacted property changes until the element is attached, which will usually ensure transactions are handled in the right order.
