MightHaveChildren

Whether the given project item might have any children at this time. This is used for project items that haven't been loaded and we don't know for certain if they have children. If it's possible that they do, then we want to return true to show that the project item can be expanded. It's a good idea to persist this property so that we can have a correct state most of the time.

ValidateName(String)

Implement this function to validate the new name. If the validation fails, this should provide a dialog to the user with the naming guidelines. This should completely cover the cases that the base validation provides, or it might otherwise provide multiple error messages to the user on different renaming attempts. For example: If your project items cannot include "Task" in their name and the user renames the item to "MyProjectItemTask" You'd provide an error dialog indicating that the users cannot include "Task" in the name, and providing the additional guidelines for renaming (e.g. invalid characters).
