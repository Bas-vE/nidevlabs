MergePersistedSettings(IUserSetting)

Called to merge persisted settings parsed from disk with any existing settings that are already present in memory. There is an inherent race condition with parsing user settings on demand as needed for a definition and updating user settings from the loading of the definition itself. Since there is no way to guarantee which one will occur first, this function provides a mechanism to retain any persisted settings that might still be important when there is a conflict. This function will only be called on settings where MultipleAllowed is set to false. If MultipleAllowed is set to true, the parsed setting will be added as another setting in the component collection and no merge will be performed. This function is called after all post parse fixups for the setting have been completed but before parse complete fixups have run.
This function is only called on the top level IUserSetting for a given IUserSettingsManager. If you have IUserSettings nested within other IUserSettings, you will need to chain the merge call as required to the nested IUserSettings.
