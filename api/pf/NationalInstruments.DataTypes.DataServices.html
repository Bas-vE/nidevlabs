AreDataValuesComparableAndEquivalent(NIType, Object, Object, out Boolean)

Tries to determine whether two value objects for the given NIType are equivalent. Like AreDataValuesEquivalent(NIType, Object, Object), this method does not verify that the supplied objects are appropriate for the supplied type, but it does attempt to handle any exceptions generated by an invalid compare and reports that the invalid pair is both not comparable and not equal.

IsImmutableType(Type)

Returns whether or not a type is immutable.

Declaration
public static bool IsImmutableType(this Type type)
Parameters
Type Name Description
Type type

The Type to test for immutability
Returns
Type Description
Boolean

True if the type is known to be immutable and False otherwise.
Remarks

Immutable types are added to this function as needed. There's no built in way to automatically determine if a type is immutable so a return value of False doesn't necessarily mean that the type isn't immutable. It just means it hasn't been added to the known list of immutable types.
