Remarks

You should only make changes to transacted properties or Owner-Component relationships (which are automatically transacted) in your rule's Execute function. Otherwise during undo/redo the non-transacted changes you made will not be undone or redone. Also, rules should not add error messages to elements during their execution (aka IMessageCollection.Add of an IMessage with MessageSeverity.Error). These messages are not persisted and if the model is left in an error state, if the model were saved then loaded, rules do not run and so would not add the errors back to the model. A type prop idea is better in this case, running on TransactionStateChange, analyzing the model to see if there are errors.
