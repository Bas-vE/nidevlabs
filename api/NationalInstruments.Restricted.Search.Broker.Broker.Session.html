public class Session : ISession
Remarks

In general, import one Session instance for each place in your UI where you want an independent query/result set. You MUST call Initialize(Guid, IDictionary<String, Object>) exactly once on each session instance. You must call all methods from the thread that initialized the session and all events are raised on that thread. Session objects enforce thread affinity which only allow to run under Dispatcher thread.
