Class SimpleDependencyExportService

Base class that can be used when model is backed by Envoy and derived class doesn't want to persist cached export information This base class assumes that derived class will need access to model to initialize itself as well as when model goes through some change Hence this base class will load the model if and when necessary only when initialized first time and derived class is given opportunity to cache the information from the model If derived class has built in persistence and has a way to verify the cached information, it can override the Initialize to hydrate its cache Assumptions (minimum requirements) for using this base class:

  1. Model is backed by Envoy
  2. Envoy.ReferenceDefinition gives the model definition
  3. Envoy.AddObserver provides notification when model is loaded and unloaded
  4. Model definition has a transaction manager which exports TransactionStateChanged event for receiving notifications when model undergoes any edits

InitExports()

Derived class should initialize/cache the necessary export information that can be supplied immediately when requested by dependencies. Default (base) implementation of Initialize will call InitExports that derived class is required to implement.

Declaration
protected abstract void InitExports()
Remarks

The purpose of this abstract method is to give the derived class opportunity to cache the information from the model. This is sometimes is necessary when derived class is not persisting the cached information. During the call of this method, the referenced definition will be in memory. However, when this method returns, that is no longer guaranteed. It is up to the caller to keep the referenced definition in memory if necessary.

RegisterFixupForFailedInitialization(Object, Predicate<IDependencyTargetExport>, Action<IDependencyTargetExport>)

This method can be called when a call to InitializeAsync(InitializationToken) has indicated an error occurred during initialization of the export. This method registers a delegate to be called back when the export has later successfully initialized. Callers of this method do not need to unregister interest - a strong reference to the registrant is not to be maintained.

Remarks

A given fixup registrant can only register one fixup. If more than one is registered, only the latest resolution handler will be run.

Once the export is initialized successfully, it will call all registered fixups. Once all fixups have been called those fixups will be removed and never called again. This is true even if the canExecuteFixup predicate returned false.
