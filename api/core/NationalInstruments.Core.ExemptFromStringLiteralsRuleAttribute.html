Class ExemptFromStringLiteralsRuleAttribute

This attribute provides a mechanism to grant exemptions to FxCop rule NI1004, which forbids the use of string literals in code. The various 'Scope' strings describe ways to define exemptions that apply to classes, methods, et. al. outside our code base, but within a specific assembly's build. The Scope value can also be used to grant an exemption for implementation vs. usage, et. al. NOTE: One can also "permanently" disable this rule by adding an AssemblyMetaData attribute of the following format to a specific assembly's AssemblyInfo.cs: [assembly: System.Reflection.AssemblyMetadata("Localize.Disabled", "true")] This should only be used specifically when there is no intention to ever localize an assembly.

Remarks

When attached to a target without specifying any named parameters, this attribute allows a method, field, or all methods / fields on a class / struct to receive string literals as arguments or be assigned a string literal value.

If no named parameters are assigned when used at the assembly / module level, it has no effect. See the documentation of the string constants above for a list of allowed values for Scope, and the meaning of the 'Target' property for specific values of Scope.

Remarks

NOTE: This Scope is only meaningful when attaching this attribute to a class or interface. NOTE: Nested types within the target type also gain the exemption granted by this attribute.

When attached to a class or interface, the suspension of the FxCop rule also applies to overridden methods on derived / implementing classes.

To grant exemptions to all methods on a class that is not available to attach the attribute to directly (e.g. items in System), declare an instance of this attribute at assembly scope and specify the class via the Target. For example, to disable this rule for System.Exception:

[assembly: NationalInstruments.Core.Restricted.ExemptFromStringLiteralsRule(ExemptFromStringLiteralsRuleAttribute.BaseClass, "System.Exception", Justification = "All methods on this class are exempt")]

If you specify Target, it must be the full name of the class including its namespace.

To grant an exemption to a class in our code base, attach this attribute directly to the class. Specifying a value for Target is not necessary.

NOTE: New methods introduced by derived classes are not exempted! The exception to this rule is that derived class constructors are also allowed to accept string literals as arguments.

Remarks

Use this to allow the use of a specific string literal value within the code. Define the value of the literal using the Target parameter.

Behaviors for specific targets to which the attribute is attached of the attribute are as follows:

Attribute Target-Resulting Behavior

  • assembly, moduleUse WITH EXTREME CAUTION! Allows all implementation code in the assembly to use the string literal specified in Target.
  • The string specified by Target may be used throughout the class / struct implementation.
  • Enum, event, Property, Parameter, delegate, ReturnValue, GenericParameterNot implemented.
  • Constructor, Method, FieldThe string specified by Target may be used in the implementation or assigned to the field

Do not lightly grant this exemption at assembly scope, as it allows the string to be used throughout the code in an entire assembly. Do so only if you are certain that not only all current instances of the string, but any future instances of that string, will not be displayed to users.

NOTE: When applied to a type, any nested types also gain the exemption.

Behaviors for specific targets of the attribute are as follows:

Attribute Target-Resulting Behavior

  • assembly, moduleDisables the rule for the entire assembly.
  • interfaceDisables the rule for anything implementing the interface.
  • No code in the class or struct is analyzed.
  • enum, event, Property, Parameter, delegate, ReturnValue, GenericParameterNot implemented.
  • Constructor, Method, FieldThe code or default values may contain / be initialized using string literals.

For this "Scope", the value of Target is not relevant. It applies to whatever it's attached to. NOTE: When inspecting a member (method, field, etc.), both containing and base types are searched for this attribute. NOTE: When applied to a type, nested types are also exempt from the rule.

Remarks

Use this to grant exemptions to calls to a method that must be exempted, but whose implementation is not available. The method's full name must be specified as Target, e.g. "Full.Namespace.To.The.Class.Method".

Behaviors for specific targets of the attribute are as follows:

Attribute Target-Resulting Behavior

  • assembly, moduleAllows any call to the specified method in the entire assembly to accept string literal arguments.
  • Allows any call made to the specified method from within the class / struct implementation to accept string literal arguments.
  • Enum, event, Property, Field, Parameter, delegate, ReturnValue, GenericParameterNot implemented.
  • Constructor, MethodAllows calls to the specified method from within the function to accept string literal arguments.

NOTE: The exemption applies to all overloads of the method. NOTE: When applied to a type, any nested types also gain the exemption.
