Class EventChainer<T>

A helper class for chaining events from one source to another. Use this class if you have a situation where you have an object that owns another, both emit the same event, and you want object A to "pass through" events from object b.

Remarks

Often you have 2 classes, maybe even 2 classes that implement the same interface with an event in it, but in any case 2 classes that emit the same event. You want an instance of class A to hold on to an instance of class B. Class B fires the event, and you want class A to see that and forward that to any clients of class A that have registered for this event.

It would be super awesome if you could just do that like this...

class B
{
    public event EventHandler SomeCoolEvent;
}

class A
{
    private B _someBThatIUse = new B();

    public event EventHandler SomeCoolEvent
    {
        add { _someBThatIUse.SomeCoolEvent += value; }
        remove { _someBThatIUse.SomeCoolEvent -= value; }
    }
}

...and that almost totally works. Problem is if a client of class A registers for class A's SomeCoolEvent, the "source" parameter of the event handler will be the private instance of class B that class A is holding on to, since we're just dumbly passing the event through. What we really want is a convenient way to do the above, but "rewrite" the source parameter to reflect that the event is coming from class A. That is what this helper does for you.

To demonstrate, let's take the above example and make it right using this class.

class B
{
    public event EventHandler SomeCoolEvent;
}

class A
{
    private B _someBThatIUse = new B();
    private EventChainer<EventArgs> _eventChainer;

    public A()
    {
        _someBThatIUse = new B();
        _eventChainer = new EventChainer<EventArgs>(this);
        _someBThatIUse.SomeCoolEvent += _eventChainer.SourceEventHandler;
    }

    public event EventHandler SomeCoolEvent
    {
        add { _eventChainer.ChainedEvent += value; }
        remove { _eventChainer.ChainedEvent -= value; }
    }
}

Not too many more lines of code, and it actually works!
