Remarks

The version strings can either be specified as a System.Version string (e.g. 2.0.0.0) or a version string that is used by our assembly file version (e.g. 2.0.0f0). The assembly file version string is composed of:

{Major}.{Upgrade}.{Maintenance}{Phase}{Build} where: {Major} is an 8 bit integer that maps to System.Version.Major {Upgrade} is a 4 bit integer that maps to System.Version.Minor {Maintenance} is a 4 bit integer that maps to System.Version.Build {Build} is a 14 bit integer {Phase} = 'd'|'a'|'b'|'f' and stands for development, alpha, beta, and final respectively. The phase is encoded as a 2 bit integer where d=0, a=1, b=2, and f=3. The Phase along with the Build number are combined to form a 16 bit integer where the phase constitutes the 15th and 16th bit. This combined 16 bit integer then maps to System.Version.Revision.
