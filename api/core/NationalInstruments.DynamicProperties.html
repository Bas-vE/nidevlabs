IParseTable

Interface used to represent a table that is used throughout parsing and generation. These tables are owned by the parser/generator and are typically discarded after the parsing/generation operation is complete. An IParseTable is useful when you want to efficiently store unique data in a single table that would otherwise be duplicated in numerous places within the persisted content.

IPropertyLocalStorageProxy

An interface which can be used to provide access to a proxy object to be used for getting and setting dynamic properties on an object. You can use this in cases where you can't inherit directly from another class which implements IPropertyLocalStorage. Instead you can contain an object which does implement that interface (such as PlatformDependencyObject) and defer to that object for your own dynamic properties. That is, the dynamic properties for your object are instead attached to an object you contain.

IXmlMutation

Interface for mutating versioned content that's persisted as XML. When creating a mutation, it's considered best practice that your mutation have no instance state data and that your mutation doesn't have any dependencies on the source model that it's mutating. In other words, you should only operate at the XML level and not have any logic that depends on the inflated model in memory. Otherwise, your mutation may not behave as intended if the logic behind your source model is changed in a later release.
