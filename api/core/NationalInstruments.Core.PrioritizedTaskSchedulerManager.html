CurrentThreadAsyncPriority

Gets the AsyncTaskPriority for the calling thread. This will return either

  1. the true priority of the current task scheduler if the tread is running code for a scheduled task, or UI thread servicing a UI task if the task has a priority
  2. WorkerHigh if the thread is not a worker thread. This is assuming that the code is in the UI thread and is in response to some user action.

StarveWhileActiveHighPriorityAsync()

Starves the current scheduler if it is not a high priority scheduler if there are currently high priority schedulers doing work. This will completely starve the current scheduler. By calling this you must ensure that this scheduler / task does not own any resources that are required by high priority schedulers to complete. This does not include own resources which can change the priority of this scheduler such as a PriorityCriticalSection. Bumping the priority of this scheduler to a priority above normal will cause this starvation to end.
