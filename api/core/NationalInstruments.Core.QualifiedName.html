Remarks

A QualifiedName is an ordered sequence of identifiers. An identifier is a string. The first identifier of the sequence is the head and the remaining parts are the tail. When a QualifiedName is used as a name for an object it generally has a single identifier. When a QualifiedName is used as a path it can be relative or absolute. An absolutely QualifiedName starts with an empty identifier.

Two QualifiedNames can be compared for equality or for a partial match. The equality comparison is a straightforward equality comparison between the sequence of identifiers in each name. A partial match occurs when one QualifiedName contains a subset of the ordered identifiers of another QualifiedName starting at the head of both QualifiedNames.

A QualifiedName can be converted back and forth between a string representation. The follow grammar describes the syntax of a QualifiedName.

.

QualifiedName := RootIdentifier? Identifiers
RootIdentifier := Separator
Identifiers := Identifier | QualifiedName Separator Identifier
Identifier := ([a-zA-Z_]|\\[^a-zA-Z_])([a-zA-Z0-9_]|\\[^a-zA-Z0-9_])+
Separator := ':'

The QualifiedName supports multiple identifiers since some named objects can have multiple identifier names. An example is a Namespace. A namespace is an object within a single name that has multiple identifiers. In general most objects are given a QualifiedName with a single identifier. The multiple identifiers of the QualifiedName come in handy when getting a relative or absolute path to an object based on the objects name and the name of the relative scopes.
