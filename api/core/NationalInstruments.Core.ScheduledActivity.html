Remarks
  • For UI thread activities, a call to DoWork should complete rather quickly. This will help the environment stay responsive. The activity manager will report UI thread activities that run for longer than .
  • DoWork is not guaranteed to be called from the same thread each time unless that is specified by ActivityResourceAffinities.
  • Only one call to DoWork for a specific instance of Activity will be made at a time.

OnCancelled()

Called when the activity has been cancelled. A cancelled activity will no longer be scheduled to call DoWork. The user should do whatever cleanup is necessary at this time.

protected virtual void OnCancelled()
Remarks
  • A call to OnCancelled should complete rather quickly. This will help the environment stay responsive.
  • OnCancelled will NOT be called while a call to DoWorkAsync is in progress.

OnCancelling()

Called when activity cancellation is started. This happens when Cancel() is called on this activity, or when the CancellationToken associated with this activity is signaled.

Declaration
protected virtual void OnCancelling()
Remarks
  • Users can override this if they need the ability to notify their activity it has been cancelled instead of requiring the activity to poll the IsCancelled.
  • This differs from the OnCancelled() method which is only called after the cancellation is complete and the async activity is not running and will not be run again.
  • This can be called from any thread. It is up to the user to implement this in a thread-safe manner.
  • When this is called, it is possible that OnDoWork is currently being called. It is also possible that OnDoWork is not currently being called.

OnComplete()

This function is the equivalent of a "finally" block for Activities. Derived classes should override this to do any operation that they need to unconditionally do at the end of OnDoWork. This function will be called in the same thread as OnDoWork. It will be called if the work completes successfully, or is cancelled, or throws an error.
