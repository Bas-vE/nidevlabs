SynchronouslyWait(Task)

Synchronously wait on a task. CAUTION: This will block the current thread. This means it can easily deadlock the UI thread if the task awaits! (see remarks)

Declaration
public static void SynchronouslyWait(this Task task)
Parameters
Type Name Description
Task task

The task to wait on
Remarks

More generally: Do not call on tasks that might switch threads, then try to await/invoke work back on your thread. Common landmine: If you call Task.Wait from the UI thread, then that task does some work on UI thread but awaits some other thread, deadlock is guaranteed because the UI thread synch context tries to execute the await continuation on the UI.
