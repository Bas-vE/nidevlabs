ReplaceTypeWithTypeExcludingInternalsOfNamedTypes(NIType, ReplaceType)

Walk a type and potentially replace itself or its inner types with another type, as defined by the delegate provided. Because it attempts to replace every type, it is less efficient than ReplaceTypeDefinitionType(NIType, ReplaceType). This function does NOT visit the Visit[Xyz]Property methods, such as VisitAttributedTypeProperty, so if there is a type used as an attribute (strange but possible), it will not be replaced. This version does NOT walk inside named types (aka typedefs). This means that if you replace A with B and there is an A inside a typedef, that A will not be replaced. It is fine, however, for A to be a typedef -- replacing the entire typedef works. Use ReplaceTypeWithTypeIncludingInternalsOfNamedTypes(NIType, ReplaceType) if you need internal replacements.

ReplaceTypeWithTypeIncludingInternalsOfNamedTypes(NIType, ReplaceType)

Walk a type and potentially replace itself or its inner types with another type, as defined by the delegate provided. Because it attempts to replace every type, it is less efficient than ReplaceTypeDefinitionType(NIType, ReplaceType). This function does NOT visit the Visit[Xyz]Property methods, such as VisitAttributedTypeProperty, so if there is a type used as an attribute (strange but possible), it will not be replaced. This function modifies the internals of named types (aka typedefs). This is generally NOT what you want to do. Named types are generally coherent things that should not be modified by arbitrary traversals, but it may be necessary when doing some compiler transformations between source model types and runtime model types. Use ReplaceTypeWithTypeExcludingInternalsOfNamedTypes(NIType, ReplaceType) to avoid this.
