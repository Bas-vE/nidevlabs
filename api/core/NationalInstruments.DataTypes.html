HasLocalizableResourcesAttribute

This attribute is meant to be applied to types to signal that they have discoverable localizable resources, which will be expected to be found in the same assembly as the class that has this attribute applied to it. The primary reason this was created was to mark control proxy types (for the control reference use case) that they have resources that can be located by a type asset provider. The kinds of resources we expect to be localized are the proxy class name itself, which, by convention has a key in the resources of the form 'ProxyName_{YourProxyClassName}' (The 'YourProxyClassName' part is identical to the string passed into this attribute), and properties that by convention the keys should have the form of '{YourProxyClassName}_{DotNetPropertyName}'. If the resource keys do not follow this convention they will not be discovered by the type asset provider for control references.

RoundMode

Defines the method of rounding (quantization) to use with a numeric computation. For consistency and brevity, "Up" refers to the direction of positive infinity, and "Down" refers to the direction of negative infinity. "Truncate" is not used since it has different meanings for different numeric formats.

Unless otherwise noted, these modes must be implemented correctly. "Correct rounding" refers to the IEEE-754 definition, where the rounded value of an operation is guaranteed to be identical to that which would be obtained by computing the operation to infinite precision and rounding.

The modes Up, Down, ToZero, and AwayFromZero are known as directed rounding modes and are useful when computing bounds, as in interval analysis (range propagation).
