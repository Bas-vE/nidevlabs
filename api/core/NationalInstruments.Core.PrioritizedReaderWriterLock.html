Class PrioritizedReaderWriterLock

A reader writer lock that honors async task priorities, can be awaited on to enter the lock but have a thread or scheduler affinity while holding the lock

public PrioritizedReaderWriterLock(bool perExecutionContext)
Parameters
Type Name Description
Boolean perExecutionContext

if set to true then an acquired lock is bound to the scheduler that was active when the lock is acquired. When false the lock is bound to the thread on which it was acquired. Use true when it is desired to support await while holding the lock. But the schedule must be single purpose only.
