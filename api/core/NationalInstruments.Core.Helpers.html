GetDetailedStringForException(Object)

Exception.ToString() does a decent job of printing out info about an exception. But it does not recurse on the Errors property of a CompositionException or the LoaderExceptions property of a ReflectionTypeLoadException. That can be incredibly useful info in diagnosing a problem. This class does recurse on such properties, as well as on InnerException (in case some InnerException is a CompositionException, etc...) If you find other exception classes for which .ToString() does not recurse appropriately on some "nested" exception, add it here for everyone's benefit.
