GetExportedFunctions<T, TMetadata>(String)

Get a collection of methods exported by MEF. Allows importing of strongly typed MEF-exported functions.

Declaration
public virtual IEnumerable<ExportedFunction<T, TMetadata>> GetExportedFunctions<T, TMetadata>(string contractName)where T : class where TMetadata : class
Parameters
Type Name Description
String contractName

MEF contract name for the export
Returns
Type Description
System.Collections.Generic.IEnumerable<ExportedFunction<T, TMetadata>>

Collection of exported functions and metadata.
Type Parameters
Name Description
T

Type of delegate used to call the method. This type must be convertible to Delegate or a runtime error with throw. It can be a Func, Action, whatever.
TMetadata

Metadata for the export
Remarks

An advantage of using this over simply using GetExports() and the like is that this method will work with any delegate type that could successfully wrap the exported method, not just the exact signature. For instance, if you are exporting a method that returns a MemoryStream, you could import it as Func<Stream>. Likewise, if you are exporting a function that takes a Stream as an argument, you could import it as Action<MemoryStream> if you want.

Gets an object that is a MEF exported type. See the documentation for CompositionContainer.GetExportedValue for a full description. This method is a direct wrapper of CompositionContainer.GetExportedValue and should only be called on exported types that have a sharing policy of CreationPolicy.NonShared. You should only call this method if you are trying to create an interface type or a type that overrides the default constructor with an importing constructor.
