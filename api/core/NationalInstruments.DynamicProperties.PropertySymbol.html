Remarks

Every property has a "symbol", which is analogous to a name of a property as defined in a C# class, or perhaps to a dependency property object you might create in WPF using System.Windows.DependencyProperty.Register A symbol encapsulates the unique identity of a property and any metadata about the property, including how to get and set it.

Depending on the property system, a client will obtain these symbols in a variety of ways. A truly static property might be obtained by an API on the object that defines it. Attached properties will be created by a factory somewhere.
