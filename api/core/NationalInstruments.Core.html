FlagsObject<TFlag>

An extensible flags enum. Types derive from this class to provide a new Flags type. In that flags type, they can define values for the flags by creating a new instance of the type. It is recommended that each new flag instance be static and readonly. If you try to create mutliple instances of FlagsObject<TFlag> derived type with the same name, an exception will be thrown.

This type provides extensibility because not all the instances of the FlagsObject<TFlag> type need to be defined in that Type. Instead, multiple other types can expose instances of the FlagsObject<TFlag> derived type.

For example:

public sealed class Colors : FlagsObject<Colors>
{
    public static readonly Colors Red = new Colors(nameof(Red));
    public static readonly Colors Blue = new Colors(nameof(Blue));

    public Colors(string flagName) : base(flagName) { }
}

public static class ExtendedColors
{
    public static readonly Colors Gold = new Colors(nameof(Gold));
    public static readonly Colors Purple = Colors.Red | Colors.Blue;
}

UniqueName

Struct that represents the logical components of a unique name: its base name and uniquifying number. This structure provides custom sorting such that numeric suffixes are treated separate from the base name such that names are first sorted alphabetically by base name and then numerically by uniquification. By default, comparisons are done with case insensitive and culture insensitive settings. For example, by default the following sort would be produced:

Channel channel 1 CHANNEL 2 channel 10 numeric NuMeRiC 3 numeric 20

If you require a different sorting, you should use the Compare function and stay away from the built in operators when performing your comparisons.
