Class WeakKeyDictionary<TKey, TValue>

A dictionary class that stores weak references to its keys (allows its keys to be garbage collected).

Remarks

Why does this class not implement System.Collections.Generic.IDictionary<TKey, TValue>? Fair question. The answer is that it's impossible to fully honor the contract of System.Collections.Generic.IDictionary<TKey, TValue>, while allowing the GC to come in and nuke keys at any time. It makes simple things like dictionary.Count == dictionary.Keys.Count() impossible to guarantee. So we're providing the narrowest interface possible for which we can provide some meaningful guarantees to the client.

This class is not intrinsically thread safe. You must provide your own thread safety if this dictionary may be written to from multiple threads or from a different thread that it's being read from.

Remarks

This operation is constant-time, which means (among other things) that we don't check the whole dictionary to see if any of the objects has been GC'ed. This means we may return a number higher than the actual number of objects that you can TryGetValue(TKey, out TValue) on, and that the number may go down over time as we internally house-clean. The guarantee you do get is that the number will never be greater than the number of keys explicitly added to the dictionary, minus the keys explicitly removed. In other words, if you add 3 keys then remove all 3, then this will return 0. If you add 3 keys and remove 2, this will return at most 1, but may return 0 if the remaining key was GC'ed and we house cleaned it internally.
