Class CompositionHost

Provides the standard implementation of the ICompositionHost interface for this next gen system. This implementation is suitable for use in your application as well as for unit tests. Refer to the documentation on the next-gen plug-in system for more information on the role of the composition host.

Creates a cached export with the information provided and adds it to the . If a cached export is found with the same value for the metadata identifier key, then the export is replaced.

Declaration
public void AddOrReplaceDynamicCompositionCacheExport(string dynamicCompositionType, string filePath, string providerId, string exportType, IDictionary<string, object> metadata, string metadataIdentifier)
Parameters
Type Name Description

String exportType

String metadataIdentifier

The metadata key that will be used to check previous cached exports for a unique identifier.
Remarks

If your metadata has an attribute like "UniqueID", this would be the metadata identifier. Then, we would use the value found for UniqueID and compare it to the previous cached exports. If a cached export with the same value for "UniqueID" is found, then that cached export would be replaced by this one.
