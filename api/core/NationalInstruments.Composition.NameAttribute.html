Remarks

Should we do the name lookup on construction, or on the first get? Pro: If ctor args only used in ctor, we don't need to keep any members other than the data of interest Con: We pay the tax in the constructor, so if you never need the value, we did the work anyway But: How often will we discover something only never to display its name to the user?

Remarks

The attribute uses the supplied resourceType's name and assembly to locate a resource class of the same name, in the 'Resources' namespace of the type's assembly. It uses LocalizationHelpers.DefaultLocalizedNameResourceKey as the key to that resource class. If a resource cannot be located, a default from NationalInstruments.Core is used -- Core.PublicResources.MissingStringResource.
