Interface IXmlMutation

Interface for mutating versioned content that's persisted as XML. When creating a mutation, it's considered best practice that your mutation have no instance state data and that your mutation doesn't have any dependencies on the source model that it's mutating. In other words, you should only operate at the XML level and not have any logic that depends on the inflated model in memory. Otherwise, your mutation may not behave as intended if the logic behind your source model is changed in a later release.

DateAdded

Returns the date/time the mutation was introduced. This is meant to be a coarse mechanism for determining when mutations contained in different namespaces are applied to versioned content that is comprised of content from multiple namespaces. Mutations are first sorted according to their version within their own namespace and then sorted across namespaces according to the date/time they were added. If two mutations belonging to different namespaces have equivalent DateAdded fields, it's assumed there is no strict ordering requirement between them and their ordering between each other is indeterminate.

Remarks

If mutations M1 and M2 are in the same namespace. It's expected that comparisons between the Version and DateAdded don't conflict. That is: If M1.Version > M2.Version is True, then M1.DateAdded >= M2.DateAdded must also be True.
If M2.Version > M1.Version is True, then M2.DateAdded >= M1.DateAdded must also be True.
If M1.DateAdded > M2.DateAdded is True, then M1.Version >= M2.Version must also be True.
If M2.DateAdded > M1.DateAdded is True, then M2.Version >= M1.Version must also be True.
Exceptions will be generated if that's not the case.

As mentioned, this provides a coarse mechanism for sorting so it's not expected that standardizing on a particular time zone is strictly required. Persisted formats generally don't change at such a rate that sorting down to the hour is required for correct functionality. However, best practice is to specify times in UTC according to the Gregorian calendar which is the default calendar used by C#.

Version

The version of the namespace at which the mutation is triggered. For Forward mutations, the mutation is triggered if the version of the persisted content is older than or equivalent to (less than or equal) this version. For Backward mutations, the mutation is triggered if the version of the persisted content is newer than (greater than) this version. Any mutations that are triggered for a given piece of versioned content are applied in sorted order within the namespace according to the Version. For Forward mutations, the mutations are applied oldest to newest. For Backward mutations, mutations are applied newest to oldest.
