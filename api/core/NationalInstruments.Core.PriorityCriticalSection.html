Class PriorityCriticalSection

Critical section that you await on to enter put in a using statement as a replacement for a lock statement instead of

lock (obj) { // Do CS code but no awaits }

do

using (await cs.WaitForTurnAsync()) { // Do CS code and other awaits } This critical section is designed to work with the PriorityTaskScheduler and will work to increase the priority of the item which owns the lock when a new item begins waiting with a higher priority. Also waiters will be released in priority order and low priority items may starve
