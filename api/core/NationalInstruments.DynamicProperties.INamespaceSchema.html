Interface INamespaceSchema

NamespaceName

Name of the namespace for the schema. By convention, namespaces should begin with a company URI followed by the name of the assembly the namespace schema resides in (optionally omitting portions of the company name if it's redundant). For example, a namespace residing in the assembly named "NationalInstruments.MyAssembly" would have a namespace name of "http://www.ni.com/MyAssembly";. The name can be longer and you can have multiple namespaces in a single assembly, but the names should all begin with the base string above to ensure uniqueness across companies and within a company. For example, to create two namespaces in MyAssembly, you could name them "http://www.ni.com/MyAssembly/Namespace1"; and "http://www.ni.com/MyAssembly/Namespace2";.

OldestCompatibleVersion

Represents the oldest version of the namespace where all content in the current version that existed in an older version can be parsed without any loss of data. This version will typically get updated as mutations are added to a namespace as mutations break compatibility with older applications. Adding new content to a namespace doesn't mandate an update to this version. If the new content is loaded from an older version of the namespace, it will attempt to load with unknown content place holders as long as this persisted version indicates it's compatible with the older namespace attempting to perform the load. If the version isn't compatible with the older version of the namespace, the load will fail.

OldestParsableVersion

Represents the oldest version of the namespace the current version will attempt to parse. This version should rarely if ever get updated as it's generally expected that old content is always loadable in a newer version of a namespace. It's expected this version will only ever get updated if there's a radical change to the persisted format such that maintaining compatibility with older content isn't practical. As a namespace ages, maintainers may also wish to update this version in order to maintain a reasonable range of version against which to test.

Version

The current version of the namespace being used. In general, namespace schemas should not cross assembly boundaries. If you think it should, you need to talk with somebody in the platform framework team to vet your use case and ensure it can be supported. The version will typically get updated as mutations are added to the namespace and the persistence format is changed. When updating the version, it's recommended that you update the major/minor fields of the version to match current assembly file major/minor version.
