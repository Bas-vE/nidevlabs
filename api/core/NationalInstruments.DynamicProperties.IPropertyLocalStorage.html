Interface IPropertyLocalStorage

An interface implemented by an object that supports internal storage of properties.

Namespace: NationalInstruments.DynamicProperties
Assembly: NationalInstruments.Core.dll
Syntax
public interface IPropertyLocalStorage
Remarks

This interface is used by the dynamic property system whose API ise defined as extension methods in DynamicPropertyExtensions. When a client tries to get, set, or enumerate properties on an object, that object is allowed to implement this interface if it is able to store properties locally. If the object does not implement this interface or cannot store a particular property symbol locally, then the system will try to attach that property via an external store.

Remarks

Any implementation is allowed to refuse to set any particular symbol, according to how its internal storage is implemented. As an example, an object might implement its storage as a set of static C# properties. The get/set methods simply set or get the values of those properties. Of course, trying to set a property that does not exist, when your backing store is completely static makes no sense. In that case, the implementation would only return true for a symbol that could be successfully set.

If this method succeeds (returns true), the implementor must guarantee that IsSet(PropertySymbol) will return true (until ClearValue(PropertySymbol) is called) and that TryGetValue(PropertySymbol, out Object) will also succeed.
