AllowInteractiveResize(Window)

Allow interactive resizing in both directions temporarily so that a user can still resize a window in either direction. This is necessary for the width/height changes made on the window to have any effect. Change the ability to resize the window back to the user selected option once the new window size has been set.

EnsureStartupLocationOnScreen(Window)

Ensures the given window will be positioned on-screen.

Declaration
public static void EnsureStartupLocationOnScreen(this Window window)
Parameters
Type Name Description
Window window

The window to position.
Remarks

The window must have a valid source (Hwnd) due to calls to the PlatformScreen APIs. The earliest safe time to call this is in an override of OnSourceInitialized().

The behavior will depend on the value of window.WindowStartupLocation:

WindowStartupLocation.Manual: Current location is checked; if not on any monitor, relocate to center on the nearest valid monitor.

WindowStartupLocation.CenterOwner: If the window itself is not on any monitor, and the owner is valid, then center the window on the monitor nearest to the owner window. If the owner is not set, then center it on the primary monitor.

WindowStartupLocation.CenterScreen: No-op - the default behavior will be used.

NOTE: This method should not be used on minimized or maximized windows. If so, this function is a no-op
