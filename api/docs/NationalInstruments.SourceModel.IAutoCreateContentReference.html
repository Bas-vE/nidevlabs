Interface IAutoCreateContentReference

Interface for a content object that will automatically create a NationalInstruments.Restricted.SourceModel.Envoys.ContentReference when the attached. When the content is detached, the content reference will be automatically removed.

Here is an example Project hierarchy with a file that uses IAutoCreateContentReference: Project NameScopingEnvoy - Target SourceFileReference - "MyFile" ID=111 SourceFile MyDefinition ... MyContent "One" ID=222 - IAutoCreateContentReference - AutoContentReferenceName = "ContRefOne", ContentReferenceShouldBeUnderNameScope = false MyContent "Two" ID=333 - IAutoCreateContentReference - AutoContentReferenceName = "ContRefTwo", ContentReferenceShouldBeUnderNameScope = true ContentReference - "ContRefOne", ContentID=222 NameScopingContentReference - "MyFile", ContentID=111 - This name is synchronized with the owning SFR ContentReference - "ContRefTwo, ContentID=333" In this situation, the ContentReferences are created when the items IAutoCreateContentReference contents are attached. This can be on load or on edit. The ContentReferences are persisted so loading SourceFile is not always necessary.
