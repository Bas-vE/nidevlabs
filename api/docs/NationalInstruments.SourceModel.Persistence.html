ExternalIdReferenceForMergeSerializer

Serializer similar to IdReferenceSerializer. The primary difference is that Content Ids from merge scripts handled by this serializer are assumed to be for content that isn't part of the merge script. As such, it doesn't register for an identifier fixup on merge. It also doesn't allow resolving to content by Id if the merge target isn't the same as the source that created the merge script or if the content found with the given Id doesn't match the content at the time the merge script was produced. If the Id fails to resolve, the reference for the property is nulled out and no error is produced. In this case, it's assumed that failing to resolve leaves the merged element in an error state where the error can still be corrected by the user from the editor.
