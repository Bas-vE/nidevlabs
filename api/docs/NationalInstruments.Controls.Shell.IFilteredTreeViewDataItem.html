Interface IFilteredTreeViewDataItem

Ther interface to be fulfilled by objects that wish to be displayed in a FilteredTreeView. This interfae is technically optional in the sense that objects not implementing it can be added to the FilteredTreeView; however, this interface is mandatory for items in the FilteredTreeView that have children or wish for searching and display behavior that is richer than simply working off the object's ToString method.

SearchContent

A set of search terms for this object when searching. In many cases, this will likely only contain a single string that is equal to the value held in DisplayContent (so that the search term for a given object will be equal to the text that is displayed for it). However, the option is provided to add additional search terms that will result in a match for this object. For instance, if we had an item representing a False Constant, we could provide search terms of "False Constant" and "Boolean Constant," and consequently filtering by "fal" or "lean" would return the False Constant as a search result. The search that is performed is case-insensitive and will match anywhere within a SearchContent string.
