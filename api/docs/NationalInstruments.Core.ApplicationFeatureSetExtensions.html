FindClosestMatchingUICultureNameOrDefault(IApplicationFeatureSet, String)

Gets the name of the CultureInfo most closely matching the CurrentUICulture from the languages supported by the application, or the Neutral Language if no match can be found.

Declaration
public static string FindClosestMatchingUICultureNameOrDefault(this IApplicationFeatureSet featureSet, string cultureInfoName)
Parameters
Type Name Description
IApplicationFeatureSet featureSet

The application feature set specifying the available languages.
System.String cultureInfoName

If null or empty, the current UI culture is used; otherwise acts as the language to find best match for.
Returns
Type Description
System.String

The best matching language, or the neutral language (en-US) if none can be found.
Remarks

Typical usages of this method involve accessing online resources for which specific language content is available, and a specific language must be defined. For example, if the user is running in a German version of Windows, but the application only supports English and Simplified Chinesese (PRC), the CurrentUICulture may be German (presuming the user has not changed it), while online resources for the product are only available in English (US) or Simplified Chinese (PRC). In such a situation, this method would return 'en-US', as it is the neutral language we specify for our applications. If the user is running with Canadian English (en-CA), and the supported languages are en-US and zh-CN, this method will identify a match of en-US because en-CA and en-US are both forms of English (en).
