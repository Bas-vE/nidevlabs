IndexGeneratorNonPersistent

NonPersistent IndexGenerators mainly deals with resource that does not take long for checking and building index. The index is expected to be up-to-date every time before searching. Each implementer should provide MEF meta before the definition. IndexMetaData("Category Id", "Category display name", "UI client Id")]

IndexGeneratorPersistent

Persistent IndexGenerators mainly deals with resource on disk that has a huge index size or takes a long time to check and update the index. The index will NOT check if it is up-to-date before search. The index is updating by a background thread or a separate process. Each implementer should provide MEF meta before the definition. IndexMetaData("Category Id", "Category display name", "UI client Id")]
