Class NullRecruiter

The NullRecruiter is type of recruiter that will throw away all recorded transaction items, and as a consequence won't notify observers of any changes as a result of the transaction items. It is primarily intended for use cases such as loading a source model, merging within a source model, or constructing a new object where any transactions are considered side effects and aren't meaningful changes in and of themselves. While the NullRecruiter doesn't record transaction items, it will still accumulate and track content Ids within its IdentifierDictionary as needed. It is up to the user of this class to clear out the dictionary where appropriate. We create one NullRecruiter per thread on an as needed basis through the use the static member NullRecruiter.Current.

DisableTransactionRecording()

Temporarily disables recording of transactions until the returned IDisposable is disposed. If called multiple times, all IDisposables must be disposed before transaction recording will resume. This function isn't thread safe. If callers dispose from a different thread than the IDisposable was created from, it's up to the caller to ensure proper usage and synchronization.
