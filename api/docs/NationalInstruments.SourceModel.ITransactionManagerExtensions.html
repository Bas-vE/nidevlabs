BeginTransactionIfNecessary(ITransactionManager, String, TransactionPurpose)

Performs the necessary logic to either: 1.) Start a new top level transaction if one hasn't been started. 2.) Start a new nested transaction if there is already an active transaction and that transaction already contains nested transactions. 3.) Do nothing if there is already an active transaction and that transaction is empty or doesn't contain nested transactions. 4.) Do nothing if the TransactionManager passed in is null. 5.) Do nothing if enlisting changes to the source model isn't allowed at this time as a result of undo/redo or model notifications.

TransactWhenPossibleAndRecommitIfNecessary(ITransactionManager, String, TransactionPurpose, Action)

Transact the given action. Use this method when you don't know the current state of the transaction system. The change will go in as soon as possible, as opposed to TransactTopLevel, which will always wait till the end of current outer transaction. This one will also attempt to recommit, if necessary, your call. PJC - this will have performance issues if called by many clients. Better be a rare thing or else we should build into transaction manager.
