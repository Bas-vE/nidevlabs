Class GridLayoutFactory

A visual layout factory to place commands in a grid. You can define a set number of rows and columns, or give specific row definitions and column definitions. Uses GridLayoutFactoryChild to know where to place the commands.

ForConfigurationPaneHeader

The default factory to use for the Configuration Pane Header. It expects five commands to be added to the context.

  1. The Command positioned in the first row and column, which spans three rows. Usually the IconCommand.
  2. The Command positioned in the first row and third column. Usually used for the label of the third command.
  3. The Command positioned in the first row and fifth column, which spans two columns. Usually the EditableLabelNameCommand for the selected element.
  4. The Command positioned in the third row and third column. Usually used for the label of the fifth command.
  5. The Command positioned in the third row and fifth column. Usually the TypeNameCommand for the selected element.

ForConfigurationPaneHeaderWithButton

The default factory to use for the Configuration Pane Header when we have type definitions involved. It expects six commands to be added to the context.
