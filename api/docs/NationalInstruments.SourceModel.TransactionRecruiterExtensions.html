EnlistNonPersistedEditorChange<T>(ITransactionRecruiter, String, T, T, PropertySetter<T>, TransactionHints)

Enlist an undo delegate for a state change in the editor that isn't related to a source model Element and as a consequence isn't retained upon reloading of the editor. In other words, the change only impacts the in memory state of the editor and won't be preserved upon a subsequent launch of the editor. This is mostly used for things that impact the project or system as a whole such as adding a folder to a project which will also create a folder in the file system. This shouldn't be used to record changes that need to be synchronized across linked contexts.

EnlistNonPersistedElementChange<T>(ITransactionRecruiter, Element, String, T, T, PropertySetter<T>, TransactionHints)

Enlist a change to an Element that won't be persisted. In other words, the change won't be retained if the Element is reloaded from disk. However, the change should be undoable while the source model is in memory. If the Element can be loaded into multiple linked contexts, you should avoid using this function unless keeping the edit synchronized across the linked contexts isn't important or meaningful.
