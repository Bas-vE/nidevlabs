Interface ITypeAssetProvider

Provides information about a type used by the editor for type visualization. Type Asset Providers are added in the provider resource dictionary plug-ins. They are keyed off of a specific NationalInstruments.DataTypes.NIType or Type Kind or various type aliases.

DomainName

Gets a domain name string for the type. This should be the C# namespace that defines the NIType. This represents the assembly, logical grouping, or component that the Help ID belongs to. In general, types should return the xml namespace of the assembly that defines them, in the format CompanyShortName + "." + ModelingNamespace e.g. NationalInstruments.LabVIEW.DataComms

HelpId

Get a help ID string for the type. Most implementers should be returning string.Empty. You should only provide more implementation if you have help topics that you want to be displayed when the user hovers on elements linked to the type this asset provider supports. If so, use this code: string memberId = [some string built from fieldPath and other private data, often the type's name]; return ContextualHelp.HelpCommonHelpers.CreateFullHelpId(DomainName, memberId, string.Empty); See CreateFullHelpId(String, String, String) for details.

DescribeType(NIType)

Gets a brief description of the given NationalInstruments.DataTypes.NIType instance. The given type should match the type for which this provider is providing assets. This text will be seen, for example, in the tip strip when hovering over the wire. It should describe the primary wire type and any details about nested types that are important for a quick view of the type. It can be multiline.
