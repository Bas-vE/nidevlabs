CreateFullHelpId(String, String, String)

With the given information, construct a help ID that conforms to the defined format of domainId[MoC]:helpId

Declaration
public static string CreateFullHelpId(string domain, string id, string moc)
Parameters
Type Name Description
System.String domain

Domain of the element. This represents the assembly, logical grouping, or namespace that the Help ID belongs to. Examples: 'NI.LabVIEW.VI:Add' 'NI.ShellCommands:PasteCommand'. If this represents the namespace, as it should for VIs, MRDs, and CDLs, it may contain colons. Examples: 'NI:Analysis:Filtering:Filter Order Estimation' For NITypes, this is just the C# namespace, e.g. "typeof(ClassThatCreatesThisNIType).Namespace".

System.String id

The ID needs to be a string unique to this entity such that there should be no collisions within a Domain. The only time a collision should happen is if you want two or more elements to have the exact same help information. Help IDs are used to look up help from the XML files associated with this Domain. The format varies based on whose ID this is. For nodes, it is the source model name. For user-functions (like VIs, MRDs, and other user MoCs), it is the function's QualifiedName. For terminals, it is the <Name>.<Terminal Identifier>. For commands, the ID is usually agreed upon by the developer and their tech writer. For type assets, it is common to use typename_fieldname.
System.String moc

The name of the model of computation (MoC) that owns the element. This is optional -- you may pass null or empty string. You only need the MoC if multiple models of computation are defined in the same Domain. This MoC name must not include a colon ':' or any square bracket characters '[' or ']'.
Returns
Type Description
System.String

The created help ID in the format domainId[MoC]:helpId
