Remarks

In some situations, Envoys are marked as not resolvable from a Dependency out of convenience. For instance, name scopes that track and stay synchronized to the name of another Envoy aren't resolvable from a Dependency by default. This generally makes things easier for clients so they only have to deal with a single result from the resolve instead of having to identify which is the names scope and which is the other envoy. However, if you wish to find the name scope by name, you can use the property to instruct the resolve to include the result.
