Interface ILocalizableContentService

This interface defines functionality useful for localization of Element and its NationalInstruments.DynamicProperties.PropertySymbols. Elements present localizable content via PropertySymbol values that have the NationalInstruments.DynamicProperties.PropertySymbolAttributes.Localizable attribute set. In many cases, this data is a simple string, but for PropertySymbols whose values are a more complex type, such as a PropertyExpression, an implementation of this interface may be used. This service also provides a way for Element to share its localizable property symbols and the values they store in localized string dictionaries.

Absolute path to the dictionary, or null if a storage path for element cannot be determined.
Remarks

NOTE: This method simply makes a dictionary path. The CultureInfo.Name property should be used to create the path. No validation of the path is done. Do not use this function to locate a string dictionary! Instead use the LocateLocalizedStringsDictionary(Element, CultureInfo) method. For an Element in a 'loose file' project, the dictionary path shall be located in a subdirectory next to the source file containing element. For an Element stored in a single file project, the path shall be in a language directory next to the single file project as elaborated in the following spec: http://ni-tfssp/tfs/LabViewDC/LabVIEW%20NextGen/Design/LV%20Express%202014/NG%20Localization%20Content%20Directory%20Structure.docx
