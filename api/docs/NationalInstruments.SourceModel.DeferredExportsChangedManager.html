Remarks

This does a couple of things. First we are breaking of the processing of deferred exports changes into multiple pieces so that we do not block the UI thread with 1 big operation. Each operation is also broken into a small piece that is scheduled with a meaningful priority. This allows user input to interleave.
And finally this gets us ready to process some exports on the background thread.
