Class XmlModelParser

This class provides a starting point for deriving a model parser for an XML format. Prior to the creation of the UniversalParser, when we had many parsers and generators, this was the home for all the common parser code. Now, all this code can eventually be merged with the UniversalParser.

RegisterParseCompleteFixup(Element, Action<Element, IElementServices>, String)

Register a function to be called when all parsing has been completed. The registered function is invoked after all elements have been parsed, all post parse fixups have run, all wiring is complete, and all elements have been attached to the source file.

The most common scenarios for using this fixup are: 1.) You have coupled fixup methods where you need to ensure one fixup runs before the other. 2.) As part of element initialization, you need to access other elements in the component hierarchy where you need to traverse through the source file in order to find them. 3.) You need to clear or reset data that is only relevant during the process of parsing.

RegisterPostParseFixup(Element, Action<Element, IElementServices>, String)

Register a function to be called when parsing is done, to perform any post-parse fixups. The registered function is invoked after all elements have been parsed and child elements have been added to their owner. While parent/child relationships have been set up, you can't assume elements have been attached yet. Elements will be attached if the parse operation is for a merge (i.e. copy/paste) and won't be attached if the parse operation is for a load. Also, at this point in time, no wiring operations have taken place.

The most common scenarios for using this fixup are: 1.) You have coupled properties (e.g. data type and value) where you need to make sure one of the properties has been parsed before you can successfully parse the other property.
2.) Performing mutation on an element when loading from older source file version. 3.) You need to access child/parent/sibling elements in the component hierarchy (minus the source file itself) as part of element initialization.
