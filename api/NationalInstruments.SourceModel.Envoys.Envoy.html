Class Envoy

Base class for an item in an EnvoyManager or one of its derived classes. An Envoy is a representative of another model item, similar to the Proxy design pattern.

ParsableNamespaceVersionCompatibility

Declaration
public override IEnumerable<ParsableNamespaceVersionCompatibility> ParsableNamespaceVersionCompatibility { get; }
Property Value
Type Description
System.Collections.Generic.IEnumerable<NationalInstruments.DynamicProperties.ParsableNamespaceVersionCompatibility>
Overrides
Element.ParsableNamespaceVersionCompatibility
Remarks

This information is used to build up the SourceModelFeatureSet when persisting model elements. The namespaces and versions returned will be combined into a single table for all Elements that are being persisted. If multiple elements require the same namespace, the highest requested version for the namespace will be stored in the table. If you don't want the element to override the NationalInstruments.DynamicProperties.INamespaceSchema.OldestCompatibleVersion, you can return VersionZero.
By default, this function will just return the namespace name defined by XmlElementName and an oldest compatible version of VersionZero. If the element can contain properties with namespaces that differ from that of the element, you should override this method to return the version information for the namespaces of the properties in addition to the version information for the element's namespace. If the element derives from an elment in another namespace or if the base class declares properties from another namespace, you should chain this call to the base class so the base class can also record its namespace and version requirements.

QueryTransientService<T>()

Performs the same functionality as QueryService except the call doesn't modify any state information on the envoy in regards to what services have been bound. Any services found are not attached and are not remembered by the envoy outside of this call. It is up to the caller to attach/detach the service if attachment is required for proper operation of the service. Unlike QueryService, multiple calls to this function will not return the same service instance. It's the responsibility of the caller to save off the return value if it needs to operate off of the same instance over an extended period of time. Calls to this function can be expensive so callers should use QueryService over this function unless they have a good reason. The most common reason for using this function is for querying services which are used to initialize the envoy and serve no purpose once the envoy has been initialized.
