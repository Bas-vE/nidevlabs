Interfaces

IDataRecordingProvider

Interface for elemnts that support data recording.

The DataDepot assembly provides the RecordingManager class which makes implementing this interface substantially easier. We recommend all implementers use it. Each IDataRecordingProvider should do the following. See VIDocument for an example of doing this.

  • Use a property backed by a Lazy RecordingManager to instantiate/access the RecordingManager as needed. You can set up the RecordingManager to handle calling StopRecording for you when time has expired for a timed recording (done via a simple .NET Timer) or manage stopping the recording yourself if you need more precise control.
  • Call Start on the RecordingManager in your StartRecording implementation asap after the recording is really started.
  • Call Finish on the RecordingManager in you StopRecording implementation asap after the recording is really done. This will call the onComplete callback provided in your StartRecording.
  • Return the RecordingManager in RecordingStatus to provide read-only information about your recording state
  • Override FinalizeCloseInternal to call base the implementation and also dispose the RecordingManager only if it has been created
